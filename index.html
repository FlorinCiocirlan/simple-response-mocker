Mock any HTTP response. In the browser.

A no-frills Chrome extension that intercepts fetch and XHR calls, matches them against rules you write, and returns whatever you want — with the status code, headers and JSON body of your choosing.

01 / how it works

Three steps. That's it.

→ step 01

Write a rule

Match by URL, glob or regex. Pick the HTTP method. Done in twelve seconds.

→ step 02

Set the response

Status code, headers, JSON body — edited in place inside the popup. No file upload, no roundtrip.

→ step 03

Ship it

Toggle the rule on. The extension intercepts matching requests and returns your mock. The page never knew.

02 / what's inside

Small surface, no surprises.

01

Request interception

Hooks fetch and XMLHttpRequest on any tab. Works against your local dev server, staging, prod — wherever the browser is pointed.

02

Per-URL rules

One rule per pattern. Each carries its own method, status code, headers and body. Switch rules on and off with a checkbox.

03

In-place JSON editor

Edit the response body right inside the extension popup. Syntax-aware, validated on save. No external editor, no copy-paste tax.

04

Custom status codes

Reproduce a 404, a 500, a flaky 502 — without convincing the backend team. Test your error states the way you'd want them tested.

03 / use it for

When the API isn't there yet.

a

The backend isn't ready

You have a contract. You don't have an endpoint. Mock the contract and keep building the UI.

b

Reproducing bugs

A user says "the page crashes when the response has no email field". Send yourself that response and watch it happen.

c

Demos and QA

Force a specific dataset on stage so the demo doesn't depend on whoever last touched the seed script.

d

Error-state design

Push 5xx and weird payloads into the page on purpose. Find out what your loading and error states actually look like.

04 / faq

Things people ask.

Is it on the Chrome Web Store yet? +
Not yet. Right now it's a Chrome extension you load unpacked from the GitHub repo. The store listing will follow once the API has stabilised.
Does it send my requests anywhere? +
No. There's no server, no telemetry, no analytics. Rules and mock bodies live in your browser's local storage. The extension does not need network access of its own.
Will it work against my real production API? +
Yes — it intercepts at the browser level, so it doesn't care what the URL points at. That said, the obvious advice applies: keep mocking turned off when you're not actively testing.
What about non-JSON responses? +
v0.1 focuses on JSON, which is the 95% case. Plain text responses work; binary and streamed responses are not supported yet.
Firefox? Safari? +
Chromium-based browsers (Chrome, Edge, Brave, Arc) should work since the extension is a Manifest V3 Chrome extension. Firefox and Safari are not on the list right now.
